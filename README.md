# Job Tracker

Локальный трекер вакансий для нескольких кандидатов. Для каждого профиля хранятся собственные критерии поиска, вакансии и отметки «Подалась». Сайт собирается в один автономный `index.html` и не требует сервера.

## Что уже настроено

- профиль `rosina` с текущими критериями и вакансиями;
- выбор кандидата на сайте;
- активные вакансии и отдельный архив;
- фильтры, приоритет, зарплата, ссылки, краткое описание, совпадения, риски и оценка вероятности интервью/оффера;
- независимые отметки «Подалась» для каждого профиля в `localStorage` браузера;
- обновление вакансий через Codex с резервной копией и автоматической проверкой результата;
- приватное хранение CV вне Git.

## Требования

- Windows PowerShell 5.1 или PowerShell 7;
- Node.js;
- установленный и авторизованный Codex CLI (`codex` должен быть доступен в `PATH`);
- интернет для поиска и проверки вакансий.

LinkedIn MCP можно использовать как дополнительный источник, но основной скрипт от него не зависит: Codex ищет по доступным job boards и карьерным страницам.

## Структура

```text
job-tracker/
  index.html                     готовый сайт
  update-jobs.ps1                основная команда
  profiles/<id>/
    profile.json                 публичное имя и заголовок
    criteria.json                критерии поиска
    jobs.json                    вакансии этого кандидата
  private-cv/<id>/cv.pdf         приватный CV, исключён из Git
  prompts/                       инструкции для Codex
  scripts/                       генератор сайта и вспомогательный код
  tests/                         автоматические проверки
  .backups/                      резервные копии, исключены из Git
```

## Открытие сайта

Откройте `index.html` двойным щелчком. Вверху можно выбрать профиль. Отметки «Подалась» сохраняются только в текущем браузере и отдельно для каждого кандидата.

## Обновление существующего профиля

Из каталога репозитория:

```powershell
powershell -ExecutionPolicy Bypass -File .\update-jobs.ps1 -Profile rosina
```

Скрипт читает приватный CV и критерии выбранного профиля, просит Codex найти и проверить подходящие вакансии, архивирует закрытые, запускает тесты и пересобирает `index.html`.

## Создание профиля из нового CV

Рекомендуемый автоматический режим:

```powershell
powershell -ExecutionPolicy Bypass -File .\update-jobs.ps1 `
  -Profile ivan-petrov `
  -CvPath "D:\CVs\Ivan-Petrov.pdf" `
  -CriteriaMode Recommended
```

`Profile` должен быть уникальным slug: только строчные латинские буквы, цифры и дефисы. Codex проанализирует CV, создаст безопасный публичный профиль, предложит критерии и выполнит первый поиск.

Ручной ввод критериев:

```powershell
powershell -ExecutionPolicy Bypass -File .\update-jobs.ps1 `
  -Profile ivan-petrov `
  -CvPath "D:\CVs\Ivan-Petrov.pdf" `
  -CriteriaMode Manual
```

Если `-CriteriaMode` не указан и критерии пусты, скрипт предложит ручной или рекомендуемый режим.

## Замена CV

Для существующего профиля передайте новый PDF. Скрипт запросит подтверждение:

```powershell
powershell -ExecutionPolicy Bypass -File .\update-jobs.ps1 `
  -Profile rosina `
  -CvPath "D:\CVs\Rosina-New.pdf"
```

Для автоматического подтверждения добавьте `-Force`. CV сохраняется как `private-cv/<profile>/cv.pdf` и не коммитится.

## Только проверка конфигурации

```powershell
powershell -ExecutionPolicy Bypass -File .\update-jobs.ps1 -Profile rosina -CheckOnly
```

Команда проверяет инструменты, файлы профиля, полноту критериев и наличие приватного CV, не выполняя поиск.

## Критерии поиска

`profiles/<id>/criteria.json` содержит:

- `locations` — допустимая география;
- `workModels` — remote, hybrid или office;
- `roles` — целевые должности;
- `seniority` — уровни позиций;
- `employmentTypes` — постоянная работа и другие допустимые типы;
- `salary` — желаемый и приемлемый net EUR в месяц;
- `languages`, `workAuthorization`, `relocation`, `excludedIndustries`.

Эти поля можно редактировать вручную. JSON должен оставаться валидным.

## Резервные копии и ошибки

Перед изменением вакансий скрипт копирует JSON выбранного профиля в `.backups/<profile>-<timestamp>`. Если Codex, тесты или сборка завершаются ошибкой, предыдущие JSON автоматически восстанавливаются.

Ручная сборка и тесты:

```powershell
node --test tests/*.test.mjs
node scripts/build-tracker.mjs
```

## Приватность и Git

В Git входят только публичные данные профилей и вакансий. Каталоги `private-cv/` и `.backups/` находятся в `.gitignore`. Не добавляйте контактные данные, адрес, фотографию или полный текст CV в `profile.json`, `criteria.json` и `jobs.json`.

Обычная публикация изменений:

```powershell
git add .
git commit -m "Update job tracker"
git push
```

Перед коммитом полезно выполнить `git status` и убедиться, что PDF отсутствует в списке.

## Частые проблемы

- `codex was not found` — установите Codex CLI и добавьте его в `PATH`.
- `Unknown profile` — проверьте ID либо создайте профиль с `-CvPath`.
- `Criteria are incomplete` — заполните обязательные массивы в `criteria.json` или повторите создание профиля.
- вакансий мало — результат зависит от открытых объявлений, доступности сайтов и строгости критериев; повторный запуск добавит новые и перенесёт закрытые в архив.
